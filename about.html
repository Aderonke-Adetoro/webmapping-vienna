The Project

About This Project

MapLens is a web mapping project that investigates the color accessibility of Vienna's U-Bahn network map. It was developed as part of the VU Web Mapping and VU Multimedia Cartography courses at TU Wien in the summer semester of 2026.

Motivation

Vienna's five U-Bahn lines are identified primarily by color: red (U1), purple (U2), orange (U3), green (U4), and brown (U6). This color-based encoding works well for people with typical trichromatic vision, but roughly 8% of the male population has some form of color vision deficiency that can make certain line pairs indistinguishable.

Rather than simply stating that "some colors look the same," this project makes the problem visible and measurable. Users can toggle between CVD types, adjust severity, and see quantified WCAG contrast ratios — turning an abstract accessibility concept into a concrete, interactive experience.

Methodology

CVD Simulation

The simulation uses the physiologically-based model published by Machado, Oliveira, and Fernandes (2009). Their 3×3 color transformation matrices are applied in two ways:

  1. Map canvas: An SVG <feColorMatrix> filter is applied to the MapLibre GL JS canvas via the CSS filter property. This transforms all rendered pixels (basemap + U-Bahn lines) as a post-compositing step — accurately reflecting how CVD affects the entire visual field, not selective elements.
  2. Legend and UI: The same 3×3 matrices are applied via JavaScript matrix multiplication to compute the simulated color for each U-Bahn line. This updates the legend swatches, which sit outside the filtered canvas and are not affected by the CSS filter.

Basemap Design

The basemap uses a deliberate grayscale palette built on OpenFreeMap vector tiles. Grayscale tones (where R = G = B) are approximately invariant under CVD color matrices — they remain gray regardless of which cone type is absent. This ensures that the colored U-Bahn lines are the dominant visual change when a CVD simulation is activated, isolating the variable under study.

Contrast Analysis

For each pair of U-Bahn line colors, the project computes the WCAG 2.2 contrast ratio using the relative luminance formula:

Contrast Ratio = (L1 + 0.05) / (L2 + 0.05)

where L1 and L2 are the relative luminances of the lighter and darker colors, respectively. Relative luminance is computed per WCAG 2.2 using the sRGB linearisation formula with gamma correction:

L = 0.2126 × Rlin + 0.7152 × Glin + 0.0722 × Blin

The threshold used is 3:1, which is the minimum required by WCAG 2.2 Success Criterion 1.4.11 ("Non-text Contrast") for graphical elements that convey meaningful information. Line pairs that fall below this threshold are flagged as failing accessibility requirements.

Severity Interpolation

The severity slider interpolates linearly between the identity matrix (normal vision, t = 0) and the full CVD matrix (complete cone absence, t = 1). Each matrix coefficient is interpolated independently. This allows users to explore the spectrum from mild anomaly to complete dichromacy, though the perceptual effect is nonlinear — most confusion becomes apparent even at moderate severity levels.

Technology Stack

  • MapLibre GL JS 4.7.1 — Open-source WebGL map rendering library for vector tiles. Chosen over Leaflet for its superior vector tile support and WebGL pipeline.
  • OpenFreeMap — Free vector tile service providing OpenStreetMap data in the Mapbox Vector Tile format. Used for the grayscale basemap.
  • SVG <feColorMatrix> — W3C-standard SVG filter primitive that applies a 4×5 (or 5×4) matrix transformation to the RGBA channels of a rendered element. Used to simulate CVD on the map canvas.
  • Vanilla HTML, CSS, JavaScript — No frameworks, no build tools, no npm. All code is hand-written as required by the course brief. External libraries (MapLibre, fonts) are loaded via CDN.

Data Sources

  • U-Bahn route and station data: GeoJSON dataset from the City of Vienna Open Data portal (data.gv.at). Contains 98 line segments and 99 station points across 5 lines (U1, U2, U3, U4, U6) in EPSG:4326 (WGS 84).
  • Basemap tiles: OpenFreeMap vector tiles (openfreemap.org), styled with a custom grayscale MapLibre style.
  • Official U-Bahn line colors: Wiener Linien corporate identity. U1: #E3000F (red), U2: #A862A4 (purple), U3: #EF7C00 (orange), U4: #00963F (green), U6: #9D6830 (brown).

Data Notes

The GeoJSON dataset contains one feature with a null value in the LINFO (line number) field: the station "Lina-Loos-Platz," which is labelled "U2 Nord" in the remarks field. This is a planned station on the future U2 extension that has not yet been assigned a line number in the dataset. It is filtered out during data loading to avoid rendering errors. No U5 data exists in the dataset, which is correct — the U5 line is under construction and not yet in service.

WCAG 2.2 Accessibility Standards

The Web Content Accessibility Guidelines (WCAG) 2.2, published by the W3C Web Accessibility Initiative, define testable success criteria for making web content accessible. The criteria most relevant to this project are:

  • SC 1.4.1 — Use of Color (Level A): Color must not be the only visual means of conveying information. Transit maps that rely solely on line color for identification do not satisfy this criterion.
  • SC 1.4.11 — Non-text Contrast (Level AA): Graphical elements required for understanding content must have a contrast ratio of at least 3:1 against adjacent colors. This is the threshold used in our contrast analysis.
  • SC 1.4.3 — Contrast (Minimum) (Level AA): Text must have a contrast ratio of at least 4.5:1 (or 3:1 for large text). While primarily about text, this criterion sets the precedent for the overall contrast requirements.
