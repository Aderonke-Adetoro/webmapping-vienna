How colour vision deficiency affects readability of the U-Bahn network map

100%

WCAG 2.1 contrast ratios between all line pairs, computed on the simulated (post-CVD) colours. Ratios below 3:1 mean two lines are effectively indistinguishable.

The technology: SVG feColorMatrix

The simulation uses the SVG <feColorMatrix> element — a filter primitive that multiplies every pixel's RGBA values by a 5×4 matrix. Applied to the map container via one CSS property:

/* Apply to the map container — everything inside transforms */ .map-wrapper { filter: url(#cvd-protanopia); } <!-- SVG filter definition --> <filter id="cvd-protanopia"> <feColorMatrix type="matrix" values="0.152 1.053 -0.205 0 0 0.115 0.786 0.099 0 0 -0.004 -0.048 1.052 0 0 0 0 0 1 0" /> </filter>

What the matrix does

Each row defines one output colour channel as a linear combination of all input channels. For protanopia, the red output row [0.152, 1.053, -0.205] means: "almost ignore red input, heavily weight green, subtract some blue." This models the biological reality that without L-cones, the brain reconstructs the missing signal from M-cone and S-cone responses.

Severity interpolation

The severity slider interpolates between the identity matrix (normal vision) and the full Machado matrix:
result[i] = identity[i] + severity × (full[i] - identity[i])
This models anomalous trichromacy — reduced but not absent cone function, which is more common than complete dichromacy.

Why this approach

The matrices come from Machado, Oliveira & Fernandes (2009), a physiologically-based model published in IEEE TVCG. Unlike CSS hue-shifts, these are derived from measured cone spectral sensitivities. The filter is applied to the container, so lines, labels, legend, and basemap all transform simultaneously and accurately.
